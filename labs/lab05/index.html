Modules 01 & 02

The Web Standards course has been very fascinating, arousing a feeling of both excitement and challenge. While it has been exciting to learn about the behind-the-scenes of designing and publishing a typical webpage, trying to make sense of the HTML structure and all the technical jargons contained therein has been very intimidating. It was not until I was able to publish my first webpage, that I began to understand and develop confidence in the course.
As I worked on my Lab 01 assignment, I learnt a very valuable lesson. I did not read the instruction correctly, so I downloaded my about-me file in my projects folder and uploaded to GitHub. By the time I realized the mistake, I did not know how to delete the file because I did not come across that in my reading assignment. However, I researched the solution on the internet and found that to delete a file on GitHub, you need to open the file on GitHub, click the three dots on the top right corner of the open file, then select delete file. I did just that and it worked. What a relief it was.

Module 03

According to w3schools.com(2024),"CSS is the fashion designer of HTML." I would argue that there is no better way to expalin the capabilities of CSS than this. It has been very fascinating learning how CSS can be used to style HTML, and exploring all the web design possibilities that it offers. I have been able to master quite a few things, such as styling elements based on "id" and "class", manipulating various text properties of elements. I am getting very familiar with CSS box rule, which explains how HTML elements relate to the web page and each other.

What I Learned in Module 03

  1. One of the interesting features that I learnt is how to select and manipulate colors on CSS. Not only does CSS suggest various color options when trying to make a selection, which is very fantastic but also when you hover over any color, it opens up the color palette or pantone. This give you the flexiblity of creating any shade of color from a very wide spectrum. I really do think that that is very amazing.
  2. Another interesting feature that I noticed on VS code is the icons at the bottom left corner of the page or window, which indicates if there errors with your code while also explaining the solution to resolving the errors. I think that is very useful feature to have because even before validating your code, it gives you an idea of the quality of your code.

Module 05

There is no doubt that it is in layouts that magic really happens in wed development. Module 5 is by far the most complex and interesting of the class till date. It took quite a while for me to figure out which method would be the best for me to actualize my design intent. At the beginning, I struggled with containerizing the different contents in my HTML file. My layout was defaulting to header, main, and footer containers even after I created several div containers for my content. So, I deleted the main container (I do not know if that was a good idea) and my layout yielded the desired result. Even after that I was struggling with how to apply either grid or flex. Eventually after watching the following YouTube videos; https://youtu.be/i1FeOOhNnwU?si=tQy54GrT33R8Ikol, https://www.youtube.com/watch?v=JYfiaSKeYhE, and https://www.youtube.com/watch?v=wsTv9y931o8&t=1029s, I was able to get a grasp of the concepts. Prof. you were absolutely right when you said it is not something that one can just grasp in just one go at it. It too countless brainstorming and trial and errors to achieve this result.

What I Learned in Module 05

  1. The first most important consideration is to determine how you want to layout your website, either horizontally or vertically. This helps you understand what layout method will best suit your purpose.
  2. Flexbox is the most suitable method for laying out your website horizontally and stack content on top of each other, whereas Grid method is most suitable if you intend to layout your content vertically and horizontally. I analyzed the website, www.thechurchofjesuschrist.org and discovered that it designed mostly with Flexbox, everything laid out horizontally.
